# Textarea

<ComponentStatus status="stable" version="1.0.0" aria="WAI-ARIA textbox (native multiline)" />

A multi-line text input with label, helper text, validation states, character counter, clear button, resize control, and auto-resize support.

```bash
npm install @luxis-ui/react
# or
yarn add @luxis-ui/react
# or
pnpm add @luxis-ui/react
```

```tsx
import { Textarea } from '@luxis-ui/react';
```

<Callout type="info" title="Migrating from another library?">
  Luxis UI `Textarea` is equivalent to MUI's `<TextField multiline>`,
  Chakra UI's `<Textarea>`, and shadcn/ui's `Textarea`.
  It renders a native `<textarea>` element for maximum accessibility and form compatibility.
</Callout>

## Before You Start

Textarea wraps the native `<textarea>` element with a consistent design language shared with Input.
Use it whenever users need to enter more than one line of plain text.

<Callout type="tip" title="Need rich formatting?">
  If users need bold, lists, or links, use `RichTextEditor` instead.
  For single-line text, use `Input`.
</Callout>

### Controlled

When to use: When a parent component needs to read or drive the value.

<Callout type="info" title="Controlled vs Uncontrolled">
  Omit `value` for uncontrolled usage. Pass `value` + `onChange` for controlled usage,
  e.g. when syncing with a form library or URL param.
</Callout>

```tsx

      <Textarea
        label="Controlled Textarea"
        value={value}

      </pre>
```

## Textarea Best Practices

<DoAndDont
  good={`// ✅ Always provide a visible label or aria-label
<Textarea label="Message" />`}
  bad={`// ❌ Relying on placeholder as the only label
<Textarea placeholder="Message" />`}
/>

<DoAndDont
  good={`// ✅ Use maxLength to enforce and communicate limits
<Textarea maxLength={300} value={text} onChange={...} />`}
  bad={`// ❌ Silently truncating in onChange without feedback
<Textarea onChange={(e) => setText(e.target.value.slice(0, 300))} />`}
/>

<DoAndDont
  good={`// ✅ Use autoResize for open-ended content
<Textarea autoResize value={notes} onChange={...} />`}
  bad={`// ❌ Setting a fixed height that cuts off content
<Textarea style={{ height: 40 }} />`}
/>

<DoAndDont
  good={`// ✅ Pair clearable with onClear in controlled mode
<Textarea clearable value={v} onClear={() => setValue('')} />`}
  bad={`// ❌ Using clearable without onClear in controlled mode
<Textarea clearable value={v} />`}
/>

## Textarea Props & API Reference

### Textarea Props

| Prop | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `label` | `string` | `undefined` | No | Label text displayed above the textarea |
| `helperText` | `string` | `undefined` | No | Helper text shown below; hidden when `error` or `success` is set |
| `error` | `string` | `undefined` | No | Error message; triggers error state; takes precedence over `helperText` |
| `success` | `string` | `undefined` | No | Success message; triggers success state |
| `size` | `'sm' \| 'md' \| 'lg'` | `theme.global.size` | No | Size of the textarea |
| `fullWidth` | `boolean` | `false` | No | If true, takes full container width |
| `clearable` | `boolean` | `false` | No | If true, shows a clear button when value is present |
| `onClear` | `() => void` | `undefined` | No | Callback fired when clear button is clicked |
| `showCount` | `boolean` | `false` | No | If true, shows a character counter; counter also appears automatically when `maxLength` is set |
| `maxLength` | `number` | `undefined` | No | Maximum character length; enforced by the browser; auto-shows the counter |
| `rows` | `number` | `4` | No | Number of visible text rows (ignored when `autoResize` is true) |
| `resize` | `'none' \| 'both' \| 'horizontal' \| 'vertical'` | `'vertical'` | No | CSS resize handle behaviour |
| `autoResize` | `boolean` | `false` | No | If true, textarea grows to fit content; overrides `resize` |
| `required` | `boolean` | `false` | No | Marks the field as required |
| `disabled` | `boolean` | `false` | No | Disables the textarea and clear button |
| `placeholder` | `string` | `undefined` | No | Placeholder text |
| `value` | `string` | `undefined` | No | Controlled value |
| `onChange` | `React.ChangeEventHandler<HTMLTextAreaElement>` | `undefined` | No | Standard change handler |
| `wrapperClassName` | `string` | `''` | No | Additional class for the outer wrapper |
| `labelClassName` | `string` | `''` | No | Additional class for the label |
| `textareaClassName` | `string` | `''` | No | Additional class for the `<textarea>` element |

Textarea also forwards all standard HTML `<textarea>` attributes (e.g. `name`, `form`, `autoComplete`, `spellCheck`, `readOnly`, `onFocus`, `onBlur`, `onKeyDown`).

## Textarea Accessibility Guide

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| <KeyboardKey>Tab</KeyboardKey> | Moves focus to the textarea |
| <KeyboardKey>Shift+Tab</KeyboardKey> | Moves focus out of the textarea |
| Standard text keys | Type, select, cut, copy, paste inside the textarea |

### ARIA Attributes

| Attribute | Value | When |
|-----------|-------|------|
| `aria-invalid` | `"true" \| undefined` | When `error` prop is set |
| `aria-required` | `"true" \| undefined` | When `required` prop is true |
| `aria-describedby` | string | Links to helper text, error, or success message IDs |

The `<textarea>` element is natively recognized as a multiline textbox by all screen readers — no extra `role` is needed.

The error paragraph carries `role="alert"` for immediate announcement. The success message uses `<output>`. The character counter uses `aria-live="polite"`.

### Accessibility Checklist

- [ ] Provide `label` on every instance (or `aria-label` if no visible label is appropriate)
- [ ] Test with keyboard — Tab in, type, Tab out
- [ ] Confirm error and success messages are announced by assistive technology
- [ ] Verify focus ring is clearly visible on the container

## Common Textarea Patterns

### Form-Integrated Textarea

```tsx
import { useState } from 'react';
import { Textarea } from '@luxis-ui/react';

export default function FormPattern() {
  const [body, setBody] = useState('');
  const [error, setError] = useState('');

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    if (!body.trim()) { setError('Body is required.'); return; }
    setError('');
    // submit...
  };

  return (
    <form onSubmit={handleSubmit}>
      <Textarea
        label="Message"
        required
        value={body}
        onChange={(e) => setBody(e.target.value)}
        error={error}
        maxLength={500}
        fullWidth
      />
      <button type="submit">Send</button>
    </form>
  );
}
```

### Textarea with react-hook-form

```tsx
import { Controller, useForm } from 'react-hook-form';
import { Textarea } from '@luxis-ui/react';

interface FormValues { description: string; }

export default function RHFPattern() {
  const { control, handleSubmit } = useForm<FormValues>({ defaultValues: { description: '' } });

  return (
    <form onSubmit={handleSubmit((data) => console.log(data))}>
      <Controller
        name="description"
        control={control}
        rules={{ required: 'Description is required' }}
        render={({ field, fieldState }) => (
          <Textarea
            label="Description"
            value={field.value}
            onChange={field.onChange}
            error={fieldState.error?.message}
            maxLength={500}
            required
          />
        )}
      />
      <button type="submit">Save</button>
    </form>
  );
}
```

## Troubleshooting Textarea

| Symptom | Wrong ❌ | Right ✅ |
|---------|---------|---------|
| Counter not showing | No `maxLength` and `showCount` is false | Set `maxLength` — counter appears automatically |
| Clear button does nothing | Using `clearable` without `onClear` in controlled mode | Always pair `clearable` with `onClear={() => setValue('')}` |
| Textarea not growing | `autoResize` not set | Add `autoResize` and provide `value` + `onChange` |
| Content cut off | Fixed `height` via inline style | Use `rows` to set initial height; let `autoResize` or user resize handle the rest |
| Screen readers silent | No `label` and no `aria-label` | Always provide an accessible name |

<Callout type="warning" title="Next.js / SSR">
  Textarea uses `useRef` and `useEffect` for auto-resize. If you are using the
  Next.js App Router, mark any component that renders it with `"use client"`.
</Callout>

### Frequently Asked Questions

**Q: Do I need both `showCount` and `maxLength` to show a counter?**
A: No. Setting `maxLength` alone is enough — the counter (`x / max`) appears automatically. Use `showCount` without `maxLength` if you want to display a plain character count with no cap.

**Q: Why isn't `onClear` resetting the value?**
A: In controlled mode (`value` prop set), you must reset state yourself inside `onClear`. Example: `onClear={() => setValue('')}`.

**Q: Can I use Textarea with Zod or Yup?**
A: Yes. Use the controlled pattern and pass the validation error string to the `error` prop. It integrates seamlessly with `react-hook-form`'s `Controller`.

**Q: How do I set a minimum height?**
A: Use the `rows` prop (e.g. `rows={6}`) to control the initial visible height. Each row is approximately one line height.

> Textarea is a Components/Textarea React component in Luxis UI for a multi-line plain-text input.

## Source
packages/luxis-ui/src/components/Textarea/Textarea.tsx

## Story
apps/storybook/src/stories/Textarea.stories.tsx

## Install
```bash
npm install @luxis-ui/react
```

## Import
```tsx
import { Textarea } from '@luxis-ui/react';
```

## Minimal usage
```tsx
import { Textarea } from '@luxis-ui/react';

export default function MinimalTextarea() {
  return <Textarea label="Message" placeholder="Enter your message..." />;
}
```

## Key props
- `label` (string): Label above the textarea
- `maxLength` (number): Enforces a character limit and auto-shows the `current / max` counter
- `showCount` (boolean, default=`false`): Shows a plain character count when no maxLength cap is needed
- `clearable` (boolean, default=`false`): Shows a × button when value is present
- `autoResize` (boolean, default=`false`): Grows to fit content automatically
- `resize` ('none'|'both'|'horizontal'|'vertical', default=`'vertical'`): CSS resize handle
- `error` (string): Triggers error state
- `rows` (number, default=`4`): Visible text rows

## When to use
Multi-line plain text entry: messages, descriptions, bios, notes, feedback.

## When NOT to use
Single-line input → use Input. Rich formatted content → use RichTextEditor.

## Accessibility
WCAG: AA
Native <textarea> — no explicit ARIA role needed.
aria-invalid on error, aria-required on required, aria-describedby for messages, aria-live counter.
