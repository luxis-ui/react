# Timeline

<ComponentStatus status="stable" version="1.0.0" aria="Sequential step list" />

A visual step-progress indicator for sequential processes and ordered events. Supports horizontal and vertical orientations, four visual variants, five color schemes, custom icons per step, clickable steps, and three sizes.

```bash
npm install @luxis-ui/react
# or
yarn add @luxis-ui/react
# or
pnpm add @luxis-ui/react
```

```tsx
import { Timeline } from '@luxis-ui/react';
import type { TimelineStep } from '@luxis-ui/react';
```

<Callout type="info" title="Migrating from another library?">
  Luxis UI `Timeline` covers the same role as MUI's `<Stepper>` / `<Step>`.
  It is a single self-contained component — no child `<Step>` wrappers needed.
</Callout>

## Before You Start

Timeline tracks progress through a sequence of steps by comparing each step's index against `currentStep`. Steps before it are **completed**, the matching index is **active**, and steps after it are **pending**. State management is left entirely to the consumer — pass the index you control.

<Callout type="tip" title="New to Luxis UI?">
  Check our Getting Started guide for overarching conventions.
</Callout>

## How to Use Timeline in React

The minimum code to render a working Timeline:

```tsx
import { Timeline } from '@luxis-ui/react';

const steps = ['Cart', 'Shipping', 'Payment', 'Confirmation'];

export default function Example() {
  return <Timeline steps={steps} currentStep={1} />;
}
```

This renders a horizontal timeline with `steps[1]` active, `steps[0]` completed with a checkmark, and `steps[2]`–`steps[3]` pending.

### Clickable Steps

Add an `onClick` handler to any `TimelineStep` to make it interactive. The step renders with a pointer cursor.

```tsx
import { useState } from 'react';
import { Timeline } from '@luxis-ui/react';
import type { TimelineStep } from '@luxis-ui/react';

const BASE_STEPS = [
  { label: 'Create Account', description: 'Sign up with your email' },
  { label: 'Verify Email',   description: 'Check your inbox' },
  { label: 'Complete Profile', description: 'Add your details' },
  { label: 'Invite Team',    description: 'Add team members' },
  { label: 'Go Live',        description: 'Start using the platform' },
];

export default function ClickableStepsExample() {
  const [active, setActive] = useState(0);

  const steps: TimelineStep[] = BASE_STEPS.map((s, i) => ({
    ...s,
    onClick: () => setActive(i),
  }));

  return (
    <div>
      <p style={{ marginBottom: 16, fontSize: 13, color: '#6b7280' }}>
        Click any step to navigate directly to it.
      </p>
      <Timeline steps={steps} currentStep={active} variant="numbered" color="primary" />
      <div style={{ marginTop: 24, padding: '16px 20px', background: '#f9fafb', borderRadius: 8, border: '1px solid #e5e7eb', fontSize: 14 }}>
        <strong>Active: {BASE_STEPS[active].label}</strong>
        <p style={{ margin: '4px 0 0', color: '#6b7280', fontSize: 13 }}>{BASE_STEPS[active].description}</p>
      </div>
    </div>
  );
}
```

### Interactive Stepper with Prev / Next

When to use: When a wizard or multi-step form drives the step progression.

<Callout type="info" title="Controlled pattern">
  Timeline does not manage steps internally. Keep `currentStep` in your own
  `useState` and increment/decrement it with buttons.
</Callout>

```tsx
import { useState } from 'react';
import { Timeline } from '@luxis-ui/react';
import type { TimelineStep } from '@luxis-ui/react';

const STEPS: TimelineStep[] = [
  { label: 'Create Account', description: 'Sign up with your email' },
  { label: 'Verify Email',   description: 'Check your inbox' },
  { label: 'Complete Profile', description: 'Add your details' },
  { label: 'Invite Team',    description: 'Add team members' },
  { label: 'Go Live',        description: 'Start using the platform' },
];

export default function InteractiveStepperExample() {
  const [step, setStep] = useState(0);
  const total = STEPS.length;

  return (
    <div>
      <Timeline
        steps={STEPS}
        currentStep={step}
        variant="numbered"
        color="primary"
      />
      <div style={{ display: 'flex', gap: 12, marginTop: 28, justifyContent: 'center', alignItems: 'center' }}>
        <button onClick={() => setStep((s) => Math.max(0, s - 1))} disabled={step === 0}>
          ← Back
        </button>
        <span style={{ fontSize: 13, color: '#6b7280' }}>Step {step + 1} of {total}</span>
        <button onClick={() => setStep((s) => Math.min(total - 1, s + 1))} disabled={step === total - 1}>
          Next →
        </button>
      </div>
      <div style={{ marginTop: 24, padding: '16px 20px', background: '#f9fafb', borderRadius: 8, fontSize: 14 }}>
        <strong>{STEPS[step].label}</strong>
        <p style={{ margin: '4px 0 0', color: '#6b7280', fontSize: 13 }}>{STEPS[step].description}</p>
      </div>
    </div>
  );
}
```

## Timeline Best Practices

<DoAndDont
  good={`// ✅ Manage currentStep externally with useState
const [step, setStep] = useState(0);
<Timeline steps={steps} currentStep={step} />`}
  bad={`// ❌ Expecting the component to manage its own step
<Timeline steps={steps} />`}
/>

<DoAndDont
  good={`// ✅ Use string arrays for simple, label-only flows
<Timeline steps={['Cart', 'Shipping', 'Payment', 'Confirm']} currentStep={1} />`}
  bad={`// ❌ Building complex objects when strings suffice
<Timeline steps={[{ label: 'Cart' }, { label: 'Shipping' }]} currentStep={1} />`}
/>

<DoAndDont
  good={`// ✅ Add onClick to each step for direct navigation
const steps = items.map((s, i) => ({ ...s, onClick: () => setStep(i) }));
<Timeline steps={steps} currentStep={active} />`}
  bad={`// ❌ Wrapping the entire component in an onClick handler
<div onClick={handleClick}><Timeline steps={steps} currentStep={active} /></div>`}
/>

<DoAndDont
  good={`// ✅ Use orientation="vertical" for long descriptions
<Timeline steps={detailedSteps} currentStep={2} orientation="vertical" />`}
  bad={`// ❌ Using horizontal layout when descriptions make it overflow
<Timeline steps={longDescriptionSteps} currentStep={2} />`}
/>

### Timeline Props

| Prop | Type | Default | Required | Description |
|------|------|---------|----------|-------------|

| `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | No | Layout direction |
| `variant` | `'default' \| 'circle' \| 'numbered' \| 'simple'` | `'default'` | No | Visual style of the step indicator |
| `color` | `'primary' \| 'success' \| 'info' \| 'warning' \| 'error'` | `'success'` | No | Color applied to active and completed nodes and connectors |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | No | Scale of the step indicators and typography |
| `showCheckmarks` | `boolean` | `true` | No | If true, completed steps show a checkmark icon instead of their indicator |
| `className` | `string` | `''` | No | Additional CSS class for the root element |
| `style` | `React.CSSProperties` | `undefined` | No | Inline styles for the root element |

### TimelineStep Interface

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `label` | `string` | **Yes** | Display label for the step |
| `description` | `string` | No | Secondary text rendered beneath the label |
| `icon` | `React.ReactNode` | No | Custom icon node; overrides all indicator styles including checkmarks and numbers |
| `onClick` | `() => void` | No | Click handler; renders the step with a pointer cursor |

### Step Status Logic

| Condition | Status |
|-----------|--------|
| `index < currentStep` | `completed` — checkmark shown (if `showCheckmarks`) |
| `index === currentStep` | `active` — highlighted with the chosen `color` |
| `index > currentStep` | `pending` — neutral / greyed out |

The connector line between step N and step N+1 fills when `N < currentStep`.

## Timeline Accessibility Guide

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| <KeyboardKey>Tab</KeyboardKey> | Moves focus through any focusable elements inside the timeline |
| <KeyboardKey>Enter</KeyboardKey> / <KeyboardKey>Space</KeyboardKey> | Activates a clickable step (when wrapped in a focusable element) |

> The `onClick` handler on `TimelineStep` adds a pointer cursor but does not automatically add `tabIndex` or keyboard handling. For full keyboard accessibility on clickable steps, consider wrapping the step label in a `<button>` via a custom render approach or ensure your surrounding stepper nav provides equivalent keyboard control.

### ARIA Attributes

Timeline relies on document reading order for accessibility. Labels and descriptions are rendered in the DOM in step order, so screen readers announce them sequentially. No explicit ARIA roles are applied to the root — the visual progress is conveyed implicitly through text content (step labels, descriptions, and the active step's visual styling).

### Accessibility Checklist

- [ ] Ensure `currentStep` reflects the true active step — screen readers rely on text cues, not visual color
- [ ] If steps are clickable, provide equivalent keyboard navigation via surrounding prev/next buttons
- [ ] Use `description` to convey important state information (dates, status text) rather than relying on color alone
- [ ] Confirm sufficient color contrast for active and completed nodes against the background

### Multi-Step Checkout Form

Used when: A form wizard drives step progression.

```tsx
import { useState } from 'react';
import { Timeline } from '@luxis-ui/react';

const CHECKOUT_STEPS = ['Cart', 'Shipping', 'Payment', 'Confirmation'];

export default function CheckoutPattern() {
  const [step, setStep] = useState(0);

  return (
    <div>
      <Timeline steps={CHECKOUT_STEPS} currentStep={step} color="primary" />
      <div style={{ marginTop: 32 }}>
        {/* Render the active form step here */}
        {step === 0 && <CartForm onNext={() => setStep(1)} />}
        {step === 1 && <ShippingForm onBack={() => setStep(0)} onNext={() => setStep(2)} />}
        {step === 2 && <PaymentForm onBack={() => setStep(1)} onNext={() => setStep(3)} />}
        {step === 3 && <Confirmation />}
      </div>
    </div>
  );
}
```

### Onboarding Wizard

Used when: Guiding a new user through a setup process with rich per-step content.

```tsx
import { useState } from 'react';
import { Timeline } from '@luxis-ui/react';
import type { TimelineStep } from '@luxis-ui/react';

const ONBOARDING: TimelineStep[] = [
  { label: 'Create Account',   description: 'Sign up with your email' },
  { label: 'Verify Email',     description: 'Check your inbox' },
  { label: 'Complete Profile', description: 'Add your details' },
  { label: 'Invite Team',      description: 'Add team members' },
  { label: 'Go Live',          description: 'Start using the platform' },
];

export default function OnboardingPattern() {
  const [step, setStep] = useState(0);

  return (
    <div style={{ maxWidth: 640 }}>
      <Timeline
        steps={ONBOARDING}
        currentStep={step}
        variant="numbered"
        color="success"
      />
      <div style={{ marginTop: 32 }}>
        <h2>{ONBOARDING[step].label}</h2>
        <p>{ONBOARDING[step].description}</p>
        <div style={{ display: 'flex', gap: 12, marginTop: 24 }}>
          {step > 0 && (
            <button onClick={() => setStep((s) => s - 1)}>← Back</button>
          )}
          {step < ONBOARDING.length - 1 && (
            <button onClick={() => setStep((s) => s + 1)}>Continue →</button>
          )}
        </div>
      </div>
    </div>
  );
}
```

## Troubleshooting Timeline

| Symptom | Wrong ❌ | Right ✅ |
|---------|---------|---------|
| All steps appear pending | Forgetting to update `currentStep` | Manage `currentStep` with `useState` and update it on navigation |
| Checkmarks not showing on completed steps | `showCheckmarks={false}` or custom `icon` overriding them | Remove `showCheckmarks={false}`; note custom `icon` always override checkmarks |
| Steps overflow in horizontal layout | Long labels on many steps | Switch to `orientation="vertical"` or shorten labels |
| Connector fills too early / late | Misunderstanding the fill logic | Connector after step N fills when `N < currentStep` (next step is at least active) |
| Custom icon not appearing | Passing a string instead of a JSX element | Pass `icon={<MyIcon />}` not `icon="myicon"` |
| Clicking step does nothing in controlled usage | `onClick` set but `currentStep` state not updated | In the `onClick` handler call your state setter: `onClick: () => setActive(i)` |

<Callout type="warning" title="Next.js / SSR">
  Timeline is a pure presentational component with no browser-specific APIs.
  It works in SSR without a `"use client"` directive unless you use interactive
  patterns (`useState`) in the same file, in which case mark that file as a
  Client Component.
</Callout>

### Frequently Asked Questions

**Q: Can I have some steps clickable and others not?**
A: Yes. Only add `onClick` to the steps you want to be interactive. Steps without `onClick` render normally without any pointer cursor.

**Q: How do I mark all steps as completed?**
A: Set `currentStep` to `steps.length` (one past the last index). All steps will be in `completed` state.

**Q: Can I use Timeline as a read-only event log?**
A: Yes. Set `currentStep` to the index of the most recent event and `orientation="vertical"`. Omit `onClick` on all steps to keep it non-interactive.

**Q: Does the `icon` prop override checkmarks on completed steps?**
A: Yes. A step-level `icon` takes precedence over all indicator logic — including the `showCheckmarks` checkmark and the `numbered` variant number.

**Q: Can I mix string steps and object steps?**
A: No. The `steps` array must be either all strings (`string[]`) or all objects (`TimelineStep[]`). TypeScript will enforce this.
